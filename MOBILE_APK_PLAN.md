# Mobile APK Plan — Consumer Redemption App

> Scope: Android APK for end-users (consumers) to scan QR codes and redeem cash rewards.
> Admin panel stays web-only.

---

## 1. Technology Recommendation

### ❌ Apache Cordova — Skip
- Aging technology, barely maintained
- Complex config, lots of plugin conflicts
- Build tooling is painful on Windows

### ✅ Capacitor (by Ionic) — Recommended
- The modern successor to Cordova — same concept, much better DX
- Works directly with **existing React + Vite** codebase — zero rewrite
- Generates a real `.apk` / `.aab` via Android Studio
- Clean native plugin system (camera, haptics, network, storage)
- Actively maintained by Ionic team (2024)

### Alternative: PWA (Progressive Web App)
- No APK at all — users tap "Add to Home Screen" in Chrome
- Lightest option, zero build complexity
- Limitation: no Play Store listing, no in-app camera QR scanner
- Good as a Phase 1 fallback

**Decision: Build with Capacitor. Keep PWA as fallback.**

## 2. What the App Covers

Only the **consumer redemption flow** — not the admin panel.

```
App Screens
├── Splash Screen          — brand logo, auto-dismiss
├── QR Scanner             — in-app camera scanner (no browser redirect needed)
├── Step 1 — Verify QR     — validates QR, shows product + brand
├── Step 2 — Confirm       — user confirms they want to claim
├── Step 3 — Mobile OTP    — enter number, receive OTP, verify
├── Step 4 — Your Name     — first-time name entry
├── Step 5 — Reward Reveal — animated amount reveal
├── Step 6 — Choose        — UPI / Wallet / Not Now
├── Step 7 — Success       — confirmation + confetti
└── My Wallet              — balance, transaction history, withdraw
```

## 3. Architecture

### Option A — Separate Mini App (Recommended)
Create `mobile/` as a standalone Vite + React app with Capacitor.
- Copy only the user-facing components from `frontend/src/pages/user/`
- Lighter bundle, no admin code shipped to consumers
- Independent release cycle

### Option B — Wrap Existing Frontend
Add Capacitor to the existing `frontend/` project.
- Route `/redeem/*` becomes the app entry point
- Admin routes are present in bundle but inaccessible without login
- Faster to set up, messier long-term

**Recommendation: Option A — separate `mobile/` app.**

## 4. Key Technical Decisions

### QR Scanning
Currently: user opens camera → scans → browser opens URL.
In the app: use **`@capacitor-community/barcode-scanner`** or **`@capacitor/camera`** for in-app scanning.
- No need to leave the app
- Faster, smoother UX
- Works offline (scan stored, submit when connected)

### API Calls
- Same backend API at `https://yourdomain.com/api`
- No changes needed to backend
- Add CORS allowlist for the app's origin

### OTP
- Same flow — static OTP in dev (`123456`), real SMS gateway in production
- Consider adding **Capacitor SMS autofill plugin** so OTP fills automatically

### Wallet & History
- Pull from `/api/users/wallet/:mobile`
- Cache last-known balance in local storage for offline display

### Offline Handling
- Show cached wallet balance when offline
- Queue scan attempts if network drops mid-flow

### App Distribution
- **Phase 1**: Direct APK download link (no Play Store needed)
- **Phase 2**: Google Play Store listing (requires developer account ₹1,750 one-time)

## 6. Build & Release Process

```
Step 1: npm run build          → Vite builds to dist/
Step 2: npx cap sync           → copies dist/ into android/
Step 3: npx cap open android   → opens Android Studio
Step 4: Build → Generate APK   → signed .apk ready to distribute
```

One-time setup: Install Android Studio + JDK 17 on dev machine.

## 7. Potential Challenges

| Challenge | Solution |
|-----------|----------|
| Camera permission on Android | Capacitor handles this via manifest auto-config |
| QR scanner plugin compatibility | Use `@capacitor-community/barcode-scanner` v3 — well maintained |
| APK size | Code-split admin routes away, Vite tree-shaking keeps it lean (~5MB) |
| HTTPS required for camera | Backend must be on HTTPS in production |
| OTP autofill | Add `@capacitor/sms-reader` plugin (Android only) |
| Back button behavior | Capacitor's `App.addListener('backButton')` handles it |
| Splash screen + icon | Use `@capacitor/splash-screen` + `@capacitor/assets` generator |

|----------|--------|--------|

| App scope | **Consumer only** | Smaller bundle, focused UX |
