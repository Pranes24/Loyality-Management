# US-052 · User — Manual QR Number Entry (Damaged QR)
**Epic:** 13 — Sequential QR Numbering  
**Priority:** P1  
**Status:** Open  
**Phase:** 3 (Backend — Manual Redemption Endpoint) + Phase 6 (Frontend — Manual Entry Screen)

---

## User Story
> As a **User**, I want to manually enter the QR code number printed on a damaged sticker, so that I can still claim my reward even when the QR image cannot be scanned.

---

## Background
Each sticker has a sequential number printed below the QR image (e.g., `No. 00012`). If the QR is physically damaged or unreadable, the user can navigate to a manual entry screen, select the organisation, type the number, and proceed through the same redemption flow.

---

## Acceptance Criteria
- [ ] Manual entry screen is accessible from the main scan page (e.g., "Can't scan? Enter number")
- [ ] User selects the organisation from a dropdown (or it auto-detects from URL)
- [ ] User enters the QR number (numeric only, no leading zeros required)
- [ ] System looks up the QR code by `(org_id, qr_number)`
- [ ] If found and valid: proceeds to OTP → redemption flow (identical to scan flow)
- [ ] If not found: show _"QR number not found for this organisation"_
- [ ] If found but already redeemed/expired: show appropriate error (same as scan errors US-018 to US-022)
- [ ] Rate limiting: max 5 incorrect attempts per IP per hour to prevent enumeration attacks
- [ ] The number entry field accepts 1–6 digit numbers (no alpha characters)

---

## API
| Method | Endpoint | Body |
|--------|----------|------|
| POST | `/api/redeem/by-number` | `{ org_code: "TESTORG", qr_number: 12 }` |

---

## Response (same shape as scan response)
```json
{
  "qr_id": "uuid-here",
  "amount": 50,
  "batch_code": "BATCH-001",
  "product_name": "Widget A",
  "org_name": "Test Org"
}
```

---

## Security
- Do NOT expose the QR UUID in the manual entry response — only return what the scan flow returns
- Rate limit by IP: 5 failed lookups per hour
- Log all manual entry attempts with IP and org_code for audit
