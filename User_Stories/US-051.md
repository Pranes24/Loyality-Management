# US-051 · Super Admin — Adjust Org QR Quota
**Epic:** 12 — QR Quota Management  
**Priority:** P2  
**Status:** Open  
**Phase:** 4 (Backend — Super Admin Quota API)

---

## User Story
> As a **Super Admin**, I want to increase or decrease an organisation's QR quota at any time, so that I can accommodate growing organisations or correct initial allocations.

---

## Acceptance Criteria
- [ ] Super admin can increase quota freely at any time
- [ ] Super admin can decrease quota, but only to a value >= current `qr_used` count
- [ ] If decrease attempt violates the constraint, API returns HTTP 400: _"Cannot set quota below used count (X QR codes already issued)"_
- [ ] Quota change takes effect immediately
- [ ] A simple audit log entry is written: `{ org_id, old_quota, new_quota, changed_by, changed_at }`
- [ ] Super admin org list shows each org's quota and used count in a column

---

## API
| Method | Endpoint | Body |
|--------|----------|------|
| PATCH | `/api/super/orgs/:id/quota` | `{ qr_quota: 75000 }` |

---

## Validation Rules
| Condition | Result |
|-----------|--------|
| `new_quota >= qr_used` | Allowed |
| `new_quota < qr_used` | HTTP 400 error |
| `new_quota <= 0` | HTTP 400 error |
| `new_quota` not an integer | HTTP 400 error |
