# US-047 · Super Admin — Set QR Quota for Organisation
**Epic:** 12 — QR Quota Management  
**Priority:** P1  
**Status:** Open  
**Phase:** 4 (Backend — Super Admin Quota API)

---

## User Story
> As a **Super Admin**, I want to assign a fixed QR code quota to each organisation, so that each org has a defined upper limit of QR codes they can ever create across all their batches.

---

## Background
Each organisation is allocated a total pool of QR codes (e.g., 50,000). This quota is set once by the super admin and can be adjusted later. The org admin cannot create more QR codes than their allocated quota — they work within this fixed pool.

---

## Acceptance Criteria
- [ ] Super admin can set `qr_quota` when creating a new organisation
- [ ] Super admin can view and edit `qr_quota` for any existing organisation
- [ ] `qr_quota` must be a positive integer (minimum 1)
- [ ] Reducing quota below current `qr_used` count is blocked with a clear error message: _"Cannot reduce quota below currently used count (X)"_
- [ ] Quota change is reflected immediately — org admin sees updated limit on next page load
- [ ] Quota is displayed on the super admin org detail page alongside `qr_used`
- [ ] Default quota for new orgs is 0 (must be explicitly set before org admin can create batches)

---

## API
| Method | Endpoint | Body |
|--------|----------|------|
| POST | `/api/super/orgs` | `{ ..., qr_quota: 50000 }` |
| PATCH | `/api/super/orgs/:id/quota` | `{ qr_quota: 75000 }` |
| GET | `/api/super/orgs/:id` | Returns `qr_quota`, `qr_used`, `qr_remaining` |

---

## Data Model
```sql
ALTER TABLE organizations ADD COLUMN qr_quota INTEGER NOT NULL DEFAULT 0;
```

---

## Edge Cases
- Quota = 0: org admin sees "No quota assigned — contact super admin" and cannot create any batch
- Quota reduced: only blocked if new value < current used count; otherwise allowed
- Quota increase: always allowed with no restriction
